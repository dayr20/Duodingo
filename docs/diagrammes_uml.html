2. Diagramme de classes

Modele de donnees MongoDB represente sous forme de classes UML avec les relations entre entites.

classDiagram
    class User {
        +String username
        +String email
        +String password
        +Number xp
        +Number level
        +Number hearts
        +Number streak
        +Date lastActivity
        +Array~String~ completedLessons
        +Array~Object~ achievements
        +Date createdAt
        +comparePassword(candidatePassword) Boolean
        +calculateLevel() Number
    }

    class Language {
        +String name
        +String slug
        +String icon
        +String color
        +String description
        +Number order
    }

    class Topic {
        +String name
        +String slug
        +String icon
        +ObjectId language
        +Number order
        +Number requiredXP
    }

    class Lesson {
        +String title
        +String description
        +ObjectId topic
        +Number order
        +Number totalXP
        +String difficulty
        +Array~Exercise~ exercises
    }

    class Exercise {
        +String type
        +String question
        +String codeSnippet
        +Array~String~ options
        +String correctAnswer
        +String explanation
        +Number xpReward
    }

    class UserProgress {
        +ObjectId user
        +ObjectId language
        +Number totalXP
        +Array~ObjectId~ completedTopics
        +Array~Object~ lessonScores
    }

    Language "1" --> "*" Topic : contient
    Topic "1" --> "*" Lesson : contient
    Lesson "1" --> "*" Exercise : compose
    User "1" --> "*" UserProgress : possede
    UserProgress "*" --> "1" Language : concerne
Relations :
- Un Language contient plusieurs Topics
- Un Topic contient plusieurs Lessons
- Une Lesson est composee de plusieurs Exercises (embarques)
- Un User a plusieurs UserProgress (un par langage)
- Les types d'exercices : qcm, fill_code, true_false, order_code

3. Diagramme de sequence - Authentification

Flux d'inscription et de connexion d'un utilisateur.

3.1 Inscription

sequenceDiagram
    actor U as Utilisateur
    participant App as App React Native
    participant API as Serveur Express
    participant DB as MongoDB

    U->>App: Saisit pseudo, email, mot de passe
    App->>API: POST /api/auth/register
    API->>DB: Verifier email unique
    alt Email deja utilise
        DB-->>API: Email existe
        API-->>App: 400 - Email deja utilise
        App-->>U: Affiche erreur
    else Email disponible
        API->>API: Hash mot de passe (bcrypt)
        API->>DB: Creer User
        DB-->>API: User cree
        API->>API: Generer JWT
        API-->>App: 201 + token + user
        App->>App: Stocker token (AsyncStorage)
        App-->>U: Redirection vers Home
    end

3.2 Connexion

sequenceDiagram
    actor U as Utilisateur
    participant App as App React Native
    participant API as Serveur Express
    participant DB as MongoDB

    U->>App: Saisit email et mot de passe
    App->>API: POST /api/auth/login
    API->>DB: Chercher User par email
    alt User non trouve
        DB-->>API: null
        API-->>App: 401 - Identifiants invalides
        App-->>U: Affiche erreur
    else User trouve
        API->>API: Comparer mot de passe (bcrypt)
        alt Mot de passe incorrect
            API-->>App: 401 - Identifiants invalides
            App-->>U: Affiche erreur
        else Mot de passe correct
            API->>API: Generer JWT
            API-->>App: 200 + token + user
            App->>App: Stocker token (AsyncStorage)
            App-->>U: Redirection vers Home
        end
    end

4. Diagramme de sequence - Deroulement d'une lecon

Parcours complet d'une lecon depuis la selection jusqu'au resultat.

sequenceDiagram
    actor U as Utilisateur
    participant App as App React Native
    participant API as Serveur Express
    participant DB as MongoDB

    U->>App: Selectionne un langage
    App->>API: GET /api/languages/:id/topics
    API->>DB: Charger topics du langage
    DB-->>API: Liste topics
    API-->>App: Topics avec requiredXP

    U->>App: Clique sur un topic debloque
    App->>API: GET /api/languages/:langId/topics/:topicId/lessons
    API->>DB: Charger lecons du topic
    DB-->>API: Liste lecons
    API-->>App: Lecons disponibles

    U->>App: Demarre une lecon
    App->>API: GET /api/lessons/:id
    API->>DB: Charger lecon + exercices
    DB-->>API: Lecon complete
    API-->>App: Exercices de la lecon

    loop Pour chaque exercice
        App-->>U: Affiche exercice (QCM/Code/V-F/Ordre)
        U->>App: Soumet sa reponse
        App->>App: Verifie reponse + calcule score
        alt Reponse correcte
            App-->>U: Feedback vert + explication
        else Reponse incorrecte
            App-->>U: Feedback rouge + bonne reponse
            App->>App: Retire 1 coeur
        end
    end

    App->>API: POST /api/lessons/:id/complete
    Note over API: score, temps, reponses
    API->>API: Calculer XP gagne
    API->>DB: Mettre a jour User (XP, streak, achievements)
    API->>DB: Mettre a jour UserProgress
    DB-->>API: Donnees mises a jour
    API-->>App: Resultat + XP + badges
    App-->>U: Ecran resultat avec animation

Stack technique :
- Frontend : React Native 0.83 + Expo SDK 55 + React Navigation 7
- Backend : Node.js + Express 5 + Mongoose
- Base de donnees : MongoDB
- Authentification : JWT (jsonwebtoken) + bcryptjs
- Stockage local : AsyncStorage
