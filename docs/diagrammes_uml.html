Language 1 ---- * Topic : contient
Topic 1 ---- * Lesson : contient
Lesson 1 ---- * Exercise : compose de (embarque)
User 1 ---- * UserProgress : possede
UserProgress * ---- 1 Language : concerne
Relations :
- Un Language contient plusieurs Topics
- Un Topic contient plusieurs Lessons
- Une Lesson est composee de plusieurs Exercises (embarques)
- Un User a plusieurs UserProgress (un par langage)
- Les types d'exercices : qcm, fill_code, true_false, order_code

3. Diagramme de sequence - Authentification

Flux d'inscription et de connexion d'un utilisateur.

3.1 Inscription

# De Vers Action
1 Utilisateur App React Native Saisit pseudo, email, mot de passe
2 App Serveur Express POST /api/auth/register →
3 Serveur MongoDB Verifier email unique
« alt » Email deja utilise : 400 - Email deja utilise → Affiche erreur
« else » Email disponible :
4 Serveur Serveur Hash mot de passe (bcrypt)
5 Serveur MongoDB Creer User
6 Serveur Serveur Generer JWT
7 Serveur App ← 201 + token + user
8 App AsyncStorage Stocker token
9 App Utilisateur Redirection vers Home

3.2 Connexion

# De Vers Action
1 Utilisateur App React Native Saisit email et mot de passe
2 App Serveur Express POST /api/auth/login →
3 Serveur MongoDB Chercher User par email
« alt » User non trouve : 401 - Identifiants invalides → Affiche erreur
« else » User trouve :
4 Serveur Serveur Comparer mot de passe (bcrypt)
« alt » Mot de passe incorrect : 401 - Identifiants invalides → Affiche erreur
« else » Mot de passe correct :
5 Serveur Serveur Generer JWT
6 Serveur App ← 200 + token + user
7 App AsyncStorage Stocker token
8 App Utilisateur Redirection vers Home

4. Diagramme de sequence - Deroulement d'une lecon

Parcours complet d'une lecon depuis la selection jusqu'au resultat.

# De Vers Action
1 Utilisateur App Selectionne un langage
2 App API GET /api/languages/:id/topics →
3 API MongoDB Charger topics du langage
4 API App ← Topics avec requiredXP
5 Utilisateur App Clique sur un topic debloque
6 App API GET /api/languages/:langId/topics/:topicId/lessons →
7 API App ← Lecons disponibles
8 Utilisateur App Demarre une lecon
9 App API GET /api/lessons/:id →
10 API App ← Exercices de la lecon
« loop » Pour chaque exercice :
11 App Utilisateur Affiche exercice (QCM / Code / V-F / Ordre)
12 Utilisateur App Soumet sa reponse
13 App App Verifie reponse + calcule score
« alt » Correct : Feedback vert + explication
« alt » Incorrect : Feedback rouge + bonne reponse, retire 1 coeur
« end loop »
14 App API POST /api/lessons/:id/complete (score, temps, reponses) →
15 API API Calculer XP gagne
16 API MongoDB Mettre a jour User (XP, streak, achievements) + UserProgress
17 API App ← Resultat + XP + badges
18 App Utilisateur Ecran resultat avec animation

5. Diagramme d'activite - Parcours utilisateur

Flux global de l'application du lancement a l'utilisation quotidienne.

# Etape Type Description Suivant
1 Lancer l'app Debut L'utilisateur ouvre l'application → 2
2 Token stocke ? Decision Verification d'un token JWT dans AsyncStorage Oui → 3 / Non → 4
3 Token valide ? Decision Validation du token cote serveur Oui → 7 / Non → 4
4 Ecran Welcome Ecran Afficher l'ecran de bienvenue → 5 ou 6
5 Inscription Ecran Formulaire d'inscription (pseudo, email, mdp) Succes → 7 / Echec → 5
6 Connexion Ecran Formulaire de connexion (email, mdp) Succes → 7 / Echec → 6
7 Ecran Home Ecran Hub principal avec choix de langage → 8, 14, 15
8 Choisir langage Action Selectionner JS, Python ou HTML/CSS → 9
9 Voir topics Ecran Liste des topics du langage choisi → 10
10 Topic debloque ? Decision Verification si l'XP requis est atteint Oui → 11 / Non → 9
11 Demarrer lecon Action Charger les exercices de la lecon → 12
12 Coeurs restants ? Decision Verifier si l'utilisateur a encore des coeurs Oui → exercice / Non → 13
13 Lecon echouee Fin Plus de coeurs, retour a la liste → 9
14 Ecran Resultat Ecran XP gagnes + Streak + Badges debloques → 7
15 Profil / Classement Ecran Voir ses stats ou le leaderboard → 7
