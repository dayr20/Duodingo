Ce diagramme presente les interactions principales entre l'utilisateur et le systeme Duodingo.

2. Diagramme de classes

Modele de donnees MongoDB represente sous forme de classes UML avec les relations entre entites.

Relations :
- Un Language contient plusieurs Topics
- Un Topic contient plusieurs Lessons
- Une Lesson est composee de plusieurs Exercises (embarques)
- Un User a plusieurs UserProgress (un par langage)
- Les types d'exercices : qcm, fill_code, true_false, order_code

6. Diagramme d'architecture technique

Vue d'ensemble de l'architecture client-serveur de Duodingo.

Stack technique :
- Frontend : React Native 0.83 + Expo SDK 55 + React Navigation 7
- Backend : Node.js + Express 5 + Mongoose
- Base de donnees : MongoDB
- Authentification : JWT (jsonwebtoken) + bcryptjs
- Stockage local : AsyncStorage
